# Rask distribusjonsguide

## 1. Installer Docker

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh ./get-docker.sh
```

## 2. Opprett prosjektmappe

```bash
mkdir /opt/archnet && cd /opt/archnet
```

## 3. Opprett konfigurasjonsfil

### 3.1 Opprett `compose.yaml`

```yaml
services:
  archnet-server:
    image: archnet/archnet-server:latest
    container_name: archnet-server
    restart: always
    networks:
      - archnet-network
    ports:
      - 8080:8080
    depends_on:
      archnet-db:
        condition: service_healthy
      archnet-cache:
        condition: service_healthy
    volumes:
      - ./config/archnet.yaml:/app/etc/archnet.yaml

  archnet-db:
    image: mysql:8.4.5
    container_name: archnet-db
    restart: always
    networks:
      - archnet-network
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: archnet
      MYSQL_USER: archnet
      MYSQL_PASSWORD: archnetpassword
    volumes:
      - ./db:/var/lib/mysql
    command:
      - --mysql-native-password=on
      - --bind-address=0.0.0.0
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost', '-uroot', '-prootpassword']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-cache:
    image: redis:7.4.5
    container_name: archnet-cache
    restart: always
    networks:
      - archnet-network
    ports:
      - 6379:6379
    volumes:
      - ./cache:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-admin:
    image: archnet/archnet-admin-web:latest
    container_name: archnet-admin-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3001:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

  archnet-user:
    image: archnet/archnet-user-web:latest
    container_name: archnet-user-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3002:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

networks:
  archnet-network:
    driver: bridge
```

## 4. Start tjenesten

```bash
docker-compose up -d
```

## Tjenesteinformasjon

### Standard tilgangsadresse

- Administrasjonspanel: `http://serverIP:3001`
- Brukerpanel: `http://serverIP:3002`
- Backend API: `http://serverIP:8080`

### Standard påloggingsinformasjon

- Administratormail: `admin@archnets.com`
- Startpassord: `password`

### Vanlige administrasjonskommandoer

```bash
# Sjekk tjenestestatus
docker-compose ps

# Sjekk tjenestelogg
docker-compose logs

# Stopp tjenesten
docker-compose down

# Start tjenesten på nytt
docker-compose up -d
```

## Viktige merknader

1. Vennligst endre standardpassordet umiddelbart etter første pålogging.
2. Det anbefales å justere konfigurasjonen i `archnet.yaml` i henhold til faktiske behov.
3. I produksjonsmiljøer, vennligst endre alle standardpassord.
4. Sørg for at serverens brannmur har åpnet de nødvendige portene.

## Teknisk støtte

Hvis du opplever problemer, vennligst sjekk:

- Om Docker-tjenesten kjører som den skal
- Om nettverkskonfigurasjonen er korrekt
- Brannmurinnstillingene på serveren

## Oppgraderingsmetode

```bash
docker-compose pull
docker-compose up -d
```

# Én-klikks distribusjon

> **Viktige merknader før distribusjon**
>
> - **Standard port for administrasjonsgrensesnittet er 8080**: Vennligst konfigurer domenenavn eller sørg for at det er en tilgjengelig IP-adresse for å fullføre installasjonen av administrasjons- og brukergrensesnittet.
> - Tjenestemodul: For mer informasjon, vennligst se [tjenestemodul](/no-NO/docs/server/config).
> - Administrasjonsmodul: For mer informasjon, vennligst se [administrasjonsmodul](/no-NO/docs/admin/config).
> - Brukermodul: For mer informasjon, vennligst se [brukermodul](/no-NO/docs/user/config).

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/ArchNets/archnet-script/refs/heads/main/install.sh)
```

```bash
bash <(wget -qO- https://raw.githubusercontent.com/ArchNets/archnet-script/refs/heads/main/install.sh)
```
