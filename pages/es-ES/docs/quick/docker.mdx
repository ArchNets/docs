# Guía de Despliegue Rápido

## 1. Instalar Docker

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh ./get-docker.sh
```

## 2. Crear el directorio del proyecto

```bash
mkdir /opt/archnet && cd /opt/archnet
```

## 3. Crear el archivo de configuración

### 3.1 Crear `compose.yaml`

```yaml
services:
  archnet-server:
    image: archnet/archnet-server:latest
    container_name: archnet-server
    restart: always
    networks:
      - archnet-network
    ports:
      - 8080:8080
    depends_on:
      archnet-db:
        condition: service_healthy
      archnet-cache:
        condition: service_healthy
    volumes:
      - ./config/archnet.yaml:/app/etc/archnet.yaml

  archnet-db:
    image: mysql:8.4.5
    container_name: archnet-db
    restart: always
    networks:
      - archnet-network
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: archnet
      MYSQL_USER: archnet
      MYSQL_PASSWORD: archnetpassword
    volumes:
      - ./db:/var/lib/mysql
    command:
      - --mysql-native-password=on
      - --bind-address=0.0.0.0
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost', '-uroot', '-prootpassword']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-cache:
    image: redis:7.4.5
    container_name: archnet-cache
    restart: always
    networks:
      - archnet-network
    ports:
      - 6379:6379
    volumes:
      - ./cache:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-admin:
    image: archnet/archnet-admin-web:latest
    container_name: archnet-admin-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3001:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

  archnet-user:
    image: archnet/archnet-user-web:latest
    container_name: archnet-user-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3002:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

networks:
  archnet-network:
    driver: bridge
```

### 3.2 Crear `archnet.yaml`

```yaml
Host: 0.0.0.0
Port: 8080
Debug: false
JwtAuth:
  AccessSecret: 1234567890
  AccessExpire: 604800
Logger:
  ServiceName: ArchNet
  Mode: console
  Encoding: plain
  TimeFormat: '2025-01-01 00:00:00.000'
  Path: logs
  Level: debug
  MaxContentLength: 0
  Compress: false
  Stat: true
  KeepDays: 0
  StackCooldownMillis: 100
  MaxBackups: 0
  MaxSize: 0
  Rotation: daily
  FileTimeFormat: 2025-01-01T00:00:00.000Z00:00
MySQL:
  Addr: archnet-db:3306
  Dbname: archnet
  Username: archnet
  Password: archnetpassword
  Config: charset=utf8mb4&parseTime=true&loc=Asia%2FShanghai
  MaxIdleConns: 10
  MaxOpenConns: 10
  SlowThreshold: 1000
Redis:
  Host: archnet-cache:6379
  Pass:
  DB: 0
Administrator:
  Password: password
  Email: admin@archnets.com
```

## 4. Iniciar el servicio

```bash
docker-compose up -d
```

## Información del Servicio

### Dirección de acceso por defecto

- Panel de administración: `http://IP_del_servidor:3001`
- Panel de usuario: `http://IP_del_servidor:3002`
- API del backend: `http://IP_del_servidor:8080`

### Información de inicio de sesión por defecto

- Correo electrónico del administrador: `admin@archnets.com`
- Contraseña inicial: `password`

### Comandos de gestión comunes

```bash
# Ver el estado del servicio
docker-compose ps

# Ver los registros del servicio
docker-compose logs

# Detener el servicio
docker-compose down

# Reiniciar el servicio
docker-compose up -d
```

## Consideraciones

1. Cambie la contraseña por defecto inmediatamente después del primer inicio de sesión.
2. Se recomienda ajustar la configuración en `archnet.yaml` según las necesidades reales.
3. En entornos de producción, cambie todas las contraseñas por defecto.
4. Asegúrese de que el firewall del servidor tenga abiertos los puertos correspondientes.

## Soporte Técnico

Si encuentra problemas, verifique:

- Si el servicio de Docker está funcionando correctamente.
- Si la configuración de red es correcta.
- La configuración del firewall del servidor.

## Método de Actualización

```bash
docker-compose pull
docker-compose up -d
```

> **Consideraciones antes del despliegue**
>
> - **El puerto por defecto del panel de administración es 8080**: Asegúrese de tener configurada la resolución de dominio o de contar con una dirección IP disponible para completar la instalación del panel de administración y del panel de usuario sin problemas.
> - Servidor: Para más información, consulte [Servidor](/es-ES/docs/server/config).
> - Panel de administración: Para más información, consulte [Panel de administración](/es-ES/docs/admin/config).
> - Panel de usuario: Para más información, consulte [Panel de usuario](/es-ES/docs/user/config).


```bash

```
