# Быстрый справочник по развертыванию

## 1. Установка Docker

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh ./get-docker.sh
```

## 2. Создание каталога проекта

```bash
mkdir /opt/archnet && cd /opt/archnet
```

## 3. Создание конфигурационного файла

### 3.1 Создание `compose.yaml`

```yaml
services:
  archnet-server:
    image: archnet/archnet-server:latest
    container_name: archnet-server
    restart: always
    networks:
      - archnet-network
    ports:
      - 8080:8080
    depends_on:
      archnet-db:
        condition: service_healthy
      archnet-cache:
        condition: service_healthy
    volumes:
      - ./config/archnet.yaml:/app/etc/archnet.yaml

  archnet-db:
    image: mysql:8.4.5
    container_name: archnet-db
    restart: always
    networks:
      - archnet-network
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: archnet
      MYSQL_USER: archnet
      MYSQL_PASSWORD: archnetpassword
    volumes:
      - ./db:/var/lib/mysql
    command:
      - --mysql-native-password=on
      - --bind-address=0.0.0.0
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost', '-uroot', '-prootpassword']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-cache:
    image: redis:7.4.5
    container_name: archnet-cache
    restart: always
    networks:
      - archnet-network
    ports:
      - 6379:6379
    volumes:
      - ./cache:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-admin:
    image: archnet/archnet-admin-web:latest
    container_name: archnet-admin-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3001:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

  archnet-user:
    image: archnet/archnet-user-web:latest
    container_name: archnet-user-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3002:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

networks:
  archnet-network:
    driver: bridge
```

### 3.2 Создание `archnet.yaml`

```yaml
Host: 0.0.0.0
Port: 8080
Debug: false

  AccessExpire: 604800

  Path: logs
  Level: debug
  MaxContentLength: 0
  Compress: false
  Stat: true
  KeepDays: 0
  StackCooldownMillis: 100
  MaxBackups: 0
  MaxSize: 0
  Rotation: daily
  FileTimeFormat: 2025-01-01T00:00:00.000Z00:00

  Addr: archnet-db:3306

  Config: charset=utf8mb4&parseTime=true&loc=Asia%2FShanghai
  MaxIdleConns: 10
  MaxOpenConns: 10

Redis:
  Host: archnet-cache:6379
  Pass:
  DB: 0

  Password: password
  Email: admin@archnets.com
```

## 4. Запуск сервиса

```bash
docker-compose up -d
```

## Информация о сервисе

### Адрес по умолчанию

- Панель администратора: `http://IP_сервера:3001`
- Панель пользователя: `http://IP_сервера:3002`
- API бэкенда: `http://IP_сервера:8080`

### Данные для входа по умолчанию

- Электронная почта администратора: `admin@archnets.com`
- Начальный пароль: `password`

### Часто используемые команды управления

```bash
# Просмотр состояния сервиса
docker-compose ps

# Просмотр логов сервиса
docker-compose logs

# Остановка сервиса
docker-compose down

# Перезапуск сервиса
docker-compose up -d
```

## Важные замечания

1. Пожалуйста, немедленно измените пароль по умолчанию после первого входа.
2. Рекомендуется настроить конфигурацию в `archnet.yaml` в соответствии с реальными потребностями.
3. В производственной среде измените все пароли по умолчанию.
4. Убедитесь, что брандмауэр сервера открыт для соответствующих портов.

## Техническая поддержка

Если возникли проблемы, проверьте:

- Работает ли служба Docker
- Правильность сетевой конфигурации
- Настройки брандмауэра сервера

## Метод обновления

```bash
docker-compose pull
docker-compose up -d
```

# Однокнопочное развертывание

> **Важные замечания перед развертыванием**
>
> - **Порт управления по умолчанию - 8080**: заранее настройте разрешение домена или убедитесь, что у вас есть доступный IP-адрес для успешной установки панели управления и панели пользователя.
> - Сервер: для получения дополнительной информации смотрите [Сервер](/ru-RU/docs/server/config).
> - Панель управления: для получения дополнительной информации смотрите [Панель управления](/ru-RU/docs/admin/config).
> - Панель пользователя: для получения дополнительной информации смотрите [Панель пользователя](/ru-RU/docs/user/config).
