# Hızlı Dağıtım Kılavuzu

## 1. Docker'ı Kurun

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh ./get-docker.sh
```

## 2. Proje Dizini Oluşturun

```bash
mkdir /opt/archnet && cd /opt/archnet
```

## 3. Konfigürasyon Dosyası Oluşturun

### 3.1 `compose.yaml` Oluşturun

```yaml
services:
  archnet-server:
    image: archnet/archnet-server:latest
    container_name: archnet-server
    restart: always
    networks:
      - archnet-network
    ports:
      - 8080:8080
    depends_on:
      archnet-db:
        condition: service_healthy
      archnet-cache:
        condition: service_healthy
    volumes:
      - ./config/archnet.yaml:/app/etc/archnet.yaml

  archnet-db:
    image: mysql:8.4.5
    container_name: archnet-db
    restart: always
    networks:
      - archnet-network
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: archnet
      MYSQL_USER: archnet
      MYSQL_PASSWORD: archnetpassword
    volumes:
      - ./db:/var/lib/mysql
    command:
      - --mysql-native-password=on
      - --bind-address=0.0.0.0
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost', '-uroot', '-prootpassword']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-cache:
    image: redis:7.4.5
    container_name: archnet-cache
    restart: always
    networks:
      - archnet-network
    ports:
      - 6379:6379
    volumes:
      - ./cache:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-admin:
    image: archnet/archnet-admin-web:latest
    container_name: archnet-admin-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3001:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

  archnet-user:
    image: archnet/archnet-user-web:latest
    container_name: archnet-user-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3002:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

networks:
  archnet-network:
    driver: bridge
```

### 3.2 `archnet.yaml` Oluşturun

```yaml
Host: 0.0.0.0

Debug: false

  AccessSecret: 1234567890
  AccessExpire: 604800
Logger:
  ServiceName: ArchNet
  Mode: console
  Encoding: plain
  TimeFormat: '2025-01-01 00:00:00.000'
  Path: logs
  Level: debug
  MaxContentLength: 0
  Compress: false
  Stat: true
  KeepDays: 0
  StackCooldownMillis: 100
  MaxBackups: 0
  MaxSize: 0
  Rotation: daily
  FileTimeFormat: 2025-01-01T00:00:00.000Z00:00
MySQL:
  Addr: archnet-db:3306
  Dbname: archnet
  Username: archnet
  Password: archnetpassword
  Config: charset=utf8mb4&parseTime=true&loc=Asia%2FShanghai
  MaxIdleConns: 10
  MaxOpenConns: 10
  SlowThreshold: 1000
Redis:
  Host: archnet-cache:6379
  Pass:
  DB: 0
Administrator:
  Password: password
  Email: admin@archnets.com
```

## 4. Servisi Başlatın

```bash
docker-compose up -d
```

## Servis Bilgileri

### Varsayılan Erişim Adresi

- Yönetici Paneli: `http://sunucuIP:3001`
- Kullanıcı Paneli: `http://sunucuIP:3002`
- Arka Uç API: `http://sunucuIP:8080`

### Varsayılan Giriş Bilgileri

- Yönetici E-posta: `admin@archnets.com`
- İlk Şifre: `password`

### Yaygın Yönetim Komutları

```bash
# Servis durumunu kontrol et
docker-compose ps

# Servis günlüklerini görüntüle
docker-compose logs

# Servisi durdur
docker-compose down

# Servisi yeniden başlat
docker-compose up -d
```

## Dikkat Edilmesi Gerekenler

1. İlk girişten sonra varsayılan şifreyi hemen değiştirin.
2. Gerçek ihtiyaçlarınıza göre `archnet.yaml` içindeki ayarları değiştirmeniz önerilir.
3. Üretim ortamında tüm varsayılan şifreleri değiştirin.
4. Sunucu güvenlik duvarının ilgili portları açtığından emin olun.

## Teknik Destek

Herhangi bir sorunla karşılaşırsanız, lütfen kontrol edin:

- Docker servisi düzgün çalışıyor mu
- Ağ yapılandırması doğru mu
- Sunucu güvenlik duvarı ayarları

## Güncelleme Yöntemi

```bash
docker-compose pull
docker-compose up -d
```

# Tek Tıkla Dağıtım

> **Dağıtım Öncesi Dikkat Edilmesi Gerekenler**
>
> - **Yönetim panelinin varsayılan portu 8080'dir**: Yönetim ve kullanıcı panelinin kurulumu için önceden alan adı çözümlemesi yapıldığından veya kullanılabilir bir IP adresinin mevcut olduğundan emin olun.
> - Sunucu: Daha fazla bilgi için [Sunucu](/tr-TR/docs/server/config) bölümüne bakın.
> - Yönetim paneli: Daha fazla bilgi için [Yönetim Paneli](/tr-TR/docs/admin/config) bölümüne bakın.
> - Kullanıcı paneli: Daha fazla bilgi için [Kullanıcı Paneli](/tr-TR/docs/user/config) bölümüne bakın.
