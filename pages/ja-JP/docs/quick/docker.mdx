# クイックデプロイガイド

## 1. Dockerのインストール

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh ./get-docker.sh
```

## 2. プロジェクトディレクトリの作成

```bash
mkdir /opt/archnet && cd /opt/archnet
```

## 3. 設定ファイルの作成

### 3.1 `compose.yaml`の作成

```yaml
services:
  archnet-server:
    image: archnet/archnet-server:latest
    container_name: archnet-server
    restart: always
    networks:
      - archnet-network
    ports:
      - 8080:8080
    depends_on:
      archnet-db:
        condition: service_healthy
      archnet-cache:
        condition: service_healthy
    volumes:
      - ./config/archnet.yaml:/app/etc/archnet.yaml

  archnet-db:
    image: mysql:8.4.5
    container_name: archnet-db
    restart: always
    networks:
      - archnet-network
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: archnet
      MYSQL_USER: archnet
      MYSQL_PASSWORD: archnetpassword
    volumes:
      - ./db:/var/lib/mysql
    command:
      - --mysql-native-password=on
      - --bind-address=0.0.0.0
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost', '-uroot', '-prootpassword']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-cache:
    image: redis:7.4.5
    container_name: archnet-cache
    restart: always
    networks:
      - archnet-network
    ports:
      - 6379:6379
    volumes:
      - ./cache:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-admin:
    image: archnet/archnet-admin-web:latest
    container_name: archnet-admin-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3001:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

  archnet-user:
    image: archnet/archnet-user-web:latest
    container_name: archnet-user-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3002:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

networks:
  archnet-network:
    driver: bridge
```

## 4. サービスの起動

```bash
docker-compose up -d
```

## サービス情報

### デフォルトのアクセスアドレス

- 管理者パネル：`http://サーバーIP:3001`
- ユーザーパネル：`http://サーバーIP:3002`
- バックエンドAPI：`http://サーバーIP:8080`

### デフォルトのログイン情報

- 管理者メール：`admin@archnet.dev`
- 初期パスワード：`password`

### よく使う管理コマンド

```bash
# サービスの状態を確認
docker-compose ps

# サービスのログを確認
docker-compose logs

# サービスを停止
docker-compose down

# サービスを再起動
docker-compose up -d
```

## 注意事項

1. 初回ログイン後は必ずデフォルトのパスワードを変更してください。
2. 実際のニーズに応じて`archnet.yaml`の設定を調整することをお勧めします。
3. 本番環境ではすべてのデフォルトパスワードを変更してください。
4. サーバーのファイアウォールが適切なポートを開放していることを確認してください。

## 技術サポート

問題が発生した場合は、以下を確認してください：

- Dockerサービスが正常に動作しているか
- ネットワーク設定が正しいか
- サーバーファイアウォールの設定

## アップグレード方法

```bash
docker-compose pull
docker-compose up -d
```

# ワンクリックデプロイ

> **デプロイ前の注意事項**
>
> - **管理端のデフォルトポートは8080です**：事前にドメインの設定を行うか、利用可能なIPアドレスを確保して、管理端とユーザ端のインストールをスムーズに行えるようにしてください。
> - サーバー側：詳細については[サーバー側](/ja-JP/docs/server/config)を参照してください。
> - 管理端：詳細については[管理端](/ja-JP/docs/admin/config)を参照してください。
> - ユーザ端：詳細については[ユーザ端](/ja-JP/docs/user/config)を参照してください。
