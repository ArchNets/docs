# Guia de Implantação Rápida

## 1. Instalar o Docker

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh ./get-docker.sh
```

## 2. Criar o diretório do projeto

```bash
mkdir /opt/archnet && cd /opt/archnet
```

## 3. Criar o arquivo de configuração

### 3.1 Criar `compose.yaml`

```yaml
services:
  archnet-server:
    image: archnet/archnet-server:latest
    container_name: archnet-server
    restart: always
    networks:
      - archnet-network
    ports:
      - 8080:8080
    depends_on:
      archnet-db:
        condition: service_healthy
      archnet-cache:
        condition: service_healthy
    volumes:
      - ./config/archnet.yaml:/app/etc/archnet.yaml

  archnet-db:
    image: mysql:8.4.5
    container_name: archnet-db
    restart: always
    networks:
      - archnet-network
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: archnet
      MYSQL_USER: archnet
      MYSQL_PASSWORD: archnetpassword
    volumes:
      - ./db:/var/lib/mysql
    command:
      - --mysql-native-password=on
      - --bind-address=0.0.0.0
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost', '-uroot', '-prootpassword']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-cache:
    image: redis:7.4.5
    container_name: archnet-cache
    restart: always
    networks:
      - archnet-network
    ports:
      - 6379:6379
    volumes:
      - ./cache:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-admin:
    image: archnet/archnet-admin-web:latest
    container_name: archnet-admin-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3001:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

  archnet-user:
    image: archnet/archnet-user-web:latest
    container_name: archnet-user-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3002:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

networks:
  archnet-network:
    driver: bridge
```

### 3.2 Criar `archnet.yaml`

```yaml
Host: 0.0.0.0
Port: 8080
Debug: false
JwtAuth:
  AccessSecret: 1234567890
  AccessExpire: 604800
Logger:
  ServiceName: ArchNet
  Mode: console
  Encoding: plain
  TimeFormat: '2025-01-01 00:00:00.000'
  Path: logs
  Level: debug
  MaxContentLength: 0
  Compress: false
  Stat: true
  KeepDays: 0
  StackCooldownMillis: 100
  MaxBackups: 0
  MaxSize: 0
  Rotation: daily
  FileTimeFormat: 2025-01-01T00:00:00.000Z00:00
MySQL:
  Addr: archnet-db:3306
  Dbname: archnet
  Username: archnet
  Password: archnetpassword
  Config: charset=utf8mb4&parseTime=true&loc=Asia%2FShanghai
  MaxIdleConns: 10
  MaxOpenConns: 10
  SlowThreshold: 1000
Redis:
  Host: archnet-cache:6379
  Pass:
  DB: 0
Administrator:
  Password: password
  Email: admin@archnet.dev
```

## 4. Iniciar o serviço

```bash
docker-compose up -d
```

## Informações do Serviço

### Endereço de Acesso Padrão

- Painel do Administrador: `http://IP_do_servidor:3001`
- Painel do Usuário: `http://IP_do_servidor:3002`
- API do Backend: `http://IP_do_servidor:8080`

### Informações de Login Padrão

- Email do Administrador: `admin@archnet.dev`
- Senha Inicial: `password`

### Comandos de Gerenciamento Comuns

```bash
# Verificar o status do serviço
docker-compose ps

# Verificar os logs do serviço
docker-compose logs

# Parar o serviço
docker-compose down

# Reiniciar o serviço
docker-compose up -d
```

## Observações

1. Após o primeiro login, altere imediatamente a senha padrão.
2. É recomendável ajustar as configurações no `archnet.yaml` de acordo com suas necessidades.
3. Em ambientes de produção, altere todas as senhas padrão.
4. Certifique-se de que o firewall do servidor tenha as portas apropriadas abertas.

## Suporte Técnico

Se você encontrar problemas, verifique:

- Se o serviço Docker está em execução corretamente
- Se a configuração da rede está correta
- As configurações do firewall do servidor

## Método de Atualização

```bash
docker-compose pull
docker-compose up -d
```

# Implantação com Um Clique

> **Observações antes da implantação**
>
> - **A porta padrão do painel de administração é 8080**: configure o DNS ou certifique-se de ter um endereço IP disponível para concluir a instalação do painel de administração e do painel do usuário.
> - Servidor: para mais informações, consulte [Servidor](/pt-BR/docs/server/config).
> - Painel de Administração: para mais informações, consulte [Painel de Administração](/pt-BR/docs/admin/config).
> - Painel do Usuário: para mais informações, consulte [Painel do Usuário](/pt-BR/docs/user/config).
