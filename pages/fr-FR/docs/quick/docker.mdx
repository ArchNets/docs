# Guide de déploiement rapide

## 1. Installer Docker

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh ./get-docker.sh
```

## 2. Créer un répertoire de projet

```bash
mkdir /opt/archnet && cd /opt/archnet
```

## 3. Créer un fichier de configuration

### 3.1 Créer `compose.yaml`

```yaml
services:
  archnet-server:
    image: archnet/archnet-server:latest
    container_name: archnet-server
    restart: always
    networks:
      - archnet-network
    ports:
      - 8080:8080
    depends_on:
      archnet-db:
        condition: service_healthy
      archnet-cache:
        condition: service_healthy
    volumes:
      - ./config/archnet.yaml:/app/etc/archnet.yaml

  archnet-db:
    image: mysql:8.4.5
    container_name: archnet-db
    restart: always
    networks:
      - archnet-network
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: archnet
      MYSQL_USER: archnet
      MYSQL_PASSWORD: archnetpassword
    volumes:
      - ./db:/var/lib/mysql
    command:
      - --mysql-native-password=on
      - --bind-address=0.0.0.0
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost', '-uroot', '-prootpassword']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-cache:
    image: redis:7.4.5
    container_name: archnet-cache
    restart: always
    networks:
      - archnet-network
    ports:
      - 6379:6379
    volumes:
      - ./cache:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-admin:
    image: archnet/archnet-admin-web:latest
    container_name: archnet-admin-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3001:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

  archnet-user:
    image: archnet/archnet-user-web:latest
    container_name: archnet-user-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3002:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

networks:
  archnet-network:
    driver: bridge
```

### 3.2 Créer `archnet.yaml`

```yaml
Host: 0.0.0.0
Port: 8080
Debug: false
JwtAuth:
  AccessSecret: 1234567890
  AccessExpire: 604800
Logger:
  ServiceName: ArchNet
  Mode: console
  Encoding: plain
  TimeFormat: '2025-01-01 00:00:00.000'
  Path: logs
  Level: debug
  MaxContentLength: 0
  Compress: false
  Stat: true
  KeepDays: 0
  StackCooldownMillis: 100
  MaxBackups: 0
  MaxSize: 0
  Rotation: daily
  FileTimeFormat: 2025-01-01T00:00:00.000Z00:00
MySQL:
  Addr: archnet-db:3306
  Dbname: archnet
  Username: archnet
  Password: archnetpassword
  Config: charset=utf8mb4&parseTime=true&loc=Asia%2FShanghai
  MaxIdleConns: 10
  MaxOpenConns: 10
  SlowThreshold: 1000
Redis:
  Host: archnet-cache:6379
  Pass:
  DB: 0
Administrator:
  Password: password
  Email: admin@archnets.com
```

## 4. Démarrer le service

```bash
docker-compose up -d
```

## Informations sur le service

### Adresse d'accès par défaut

- Panneau d'administration : `http://IP_du_serveur:3001`
- Panneau utilisateur : `http://IP_du_serveur:3002`
- API backend : `http://IP_du_serveur:8080`

### Informations de connexion par défaut

- Email de l'administrateur : `admin@archnets.com`
- Mot de passe initial : `password`

### Commandes de gestion courantes

```bash
# Vérifier l'état des services
docker-compose ps

# Voir les journaux des services
docker-compose logs

# Arrêter les services
docker-compose down

# Redémarrer les services
docker-compose up -d
```

## Remarques

1. Veuillez changer le mot de passe par défaut immédiatement après la première connexion.
2. Il est conseillé d'ajuster la configuration dans `archnet.yaml` en fonction des besoins réels.
3. Dans un environnement de production, veuillez changer tous les mots de passe par défaut.
4. Assurez-vous que le pare-feu du serveur a ouvert les ports appropriés.

## Support technique

En cas de problème, veuillez vérifier :

- Si le service Docker fonctionne correctement
- Si la configuration réseau est correcte
- Les paramètres du pare-feu du serveur

## Méthode de mise à jour

```bash
docker-compose pull
docker-compose up -d
```

# Déploiement en un clic

> **Remarques avant le déploiement**
>
> - **Le port par défaut de l'interface d'administration est 8080** : Veuillez configurer à l'avance la résolution de nom de domaine ou vous assurer d'avoir une adresse IP disponible pour réussir l'installation de l'interface d'administration et de l'interface utilisateur.
> - Serveur : Pour plus d'informations, veuillez consulter [Serveur](/fr-FR/docs/server/config).
> - Interface d'administration : Pour plus d'informations, veuillez consulter [Administration](/fr-FR/docs/admin/config).
> - Interface utilisateur : Pour plus d'informations, veuillez consulter [Utilisateur](/fr-FR/docs/user/config).

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/ArchNets/archnet-script/refs/heads/main/install.sh)
```

## Option 2 : Utiliser `wget`

```bash
bash <(wget -qO- https://raw.githubusercontent.com/ArchNets/archnet-script/refs/heads/main/install.sh)
```
