# Швидкий посібник з розгортання

## 1. Встановлення Docker

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh ./get-docker.sh
```

## 2. Створення каталогу проекту

```bash
mkdir /opt/archnet && cd /opt/archnet
```

## 3. Створення файлу конфігурації

### 3.1 Створення `compose.yaml`

```yaml
services:
  archnet-server:
    image: archnet/archnet-server:latest
    container_name: archnet-server
    restart: always
    networks:
      - archnet-network
    ports:
      - 8080:8080
    depends_on:
      archnet-db:
        condition: service_healthy
      archnet-cache:
        condition: service_healthy
    volumes:
      - ./config/archnet.yaml:/app/etc/archnet.yaml

  archnet-db:
    image: mysql:8.4.5
    container_name: archnet-db
    restart: always
    networks:
      - archnet-network
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: archnet
      MYSQL_USER: archnet
      MYSQL_PASSWORD: archnetpassword
    volumes:
      - ./db:/var/lib/mysql
    command:
      - --mysql-native-password=on
      - --bind-address=0.0.0.0
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost', '-uroot', '-prootpassword']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-cache:
    image: redis:7.4.5
    container_name: archnet-cache
    restart: always
    networks:
      - archnet-network
    ports:
      - 6379:6379
    volumes:
      - ./cache:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-admin:
    image: archnet/archnet-admin-web:latest
    container_name: archnet-admin-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3001:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

  archnet-user:
    image: archnet/archnet-user-web:latest
    container_name: archnet-user-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3002:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

networks:
  archnet-network:
    driver: bridge
```

## 4. Запуск сервісу

```bash
docker-compose up -d
```

## Інформація про сервіс

### За замовчуванням адреса доступу

- Панель адміністратора: `http://IP_сервера:3001`
- Панель користувача: `http://IP_сервера:3002`
- API бекенду: `http://IP_сервера:8080`

### За замовчуванням дані для входу

- Email адміністратора: `admin@archnets.com`
- Початковий пароль: `password`

### Загальні команди управління

```bash
# Перегляд стану сервісу
docker-compose ps

# Перегляд журналів сервісу
docker-compose logs

# Зупинка сервісу
docker-compose down

# Перезапуск сервісу
docker-compose up -d
```

## Зверніть увагу

1. Після першого входу терміново змініть пароль за замовчуванням
2. Рекомендується налаштувати конфігурацію в `archnet.yaml` відповідно до ваших потреб
3. У виробничому середовищі змініть усі паролі за замовчуванням
4. Переконайтеся, що брандмауер сервера відкритий для відповідних портів

## Технічна підтримка

Якщо виникли проблеми, перевірте:

- Чи працює служба Docker
- Чи правильно налаштовано мережу
- Налаштування брандмауера сервера

## Метод оновлення

```bash
docker-compose pull
docker-compose up -d
```

# Одноклікове розгортання

> **Зверніть увагу перед розгортанням**
>
> - **Порт адміністративної панелі за замовчуванням - 8080**: будь ласка, заздалегідь налаштуйте DNS або переконайтеся, що у вас є доступна IP-адреса для успішного завершення установки адміністративної та користувацької панелей.
> - Сервер: більше інформації дивіться в [Сервері](/uk-UA/docs/server/config).
> - Адміністративна панель: більше інформації дивіться в [Адміністративній панелі](/uk-UA/docs/admin/config).
> - Користувацька панель: більше інформації дивіться в [Користувацькій панелі](/uk-UA/docs/user/config).

## Варіант 1: Використання `curl`

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/ArchNets/archnet-script/refs/heads/main/install.sh)
```

## Варіант 2: Використання `wget`

```bash
bash <(wget -qO- https://raw.githubusercontent.com/ArchNets/archnet-script/refs/heads/main/install.sh)
```
