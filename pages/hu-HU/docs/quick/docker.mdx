# Gyors telepítési útmutató

## 1. Docker telepítése

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh ./get-docker.sh
```

## 2. Projekt könyvtár létrehozása

```bash
mkdir /opt/archnet && cd /opt/archnet
```

## 3. Konfigurációs fájl létrehozása

### 3.1 `compose.yaml` létrehozása

```yaml
services:
  archnet-server:
    image: archnet/archnet-server:latest
    container_name: archnet-server
    restart: always
    networks:
      - archnet-network
    ports:
      - 8080:8080
    depends_on:
      archnet-db:
        condition: service_healthy
      archnet-cache:
        condition: service_healthy
    volumes:
      - ./config/archnet.yaml:/app/etc/archnet.yaml

  archnet-db:
    image: mysql:8.4.5
    container_name: archnet-db
    restart: always
    networks:
      - archnet-network
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: archnet
      MYSQL_USER: archnet
      MYSQL_PASSWORD: archnetpassword
    volumes:
      - ./db:/var/lib/mysql
    command:
      - --mysql-native-password=on
      - --bind-address=0.0.0.0
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost', '-uroot', '-prootpassword']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-cache:
    image: redis:7.4.5
    container_name: archnet-cache
    restart: always
    networks:
      - archnet-network
    ports:
      - 6379:6379
    volumes:
      - ./cache:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-admin:
    image: archnet/archnet-admin-web:latest
    container_name: archnet-admin-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3001:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

  archnet-user:
    image: archnet/archnet-user-web:latest
    container_name: archnet-user-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3002:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

networks:
  archnet-network:
    driver: bridge
```

## 4. Szolgáltatás indítása

```bash
docker-compose up -d
```

## Szolgáltatás információk

### Alapértelmezett elérhetőség

- Adminisztrátori panel: `http://szerverIP:3001`
- Felhasználói panel: `http://szerverIP:3002`
- Háttér API: `http://szerverIP:8080`

### Alapértelmezett bejelentkezési információk

- Adminisztrátor email: `admin@archnet.dev`
- Kezdeti jelszó: `password`

### Gyakran használt adminisztrációs parancsok

```bash
# Szolgáltatás állapotának megtekintése
docker-compose ps

# Szolgáltatás naplóinak megtekintése
docker-compose logs

# Szolgáltatás leállítása
docker-compose down

# Szolgáltatás újraindítása
docker-compose up -d
```

## Fontos megjegyzések

1. Első bejelentkezés után azonnal módosítsa az alapértelmezett jelszót.
2. Javasolt a `archnet.yaml` konfigurációs fájl beállításainak módosítása a tényleges igények szerint.
3. Termelési környezetben minden alapértelmezett jelszót módosítson.
4. Győződjön meg arról, hogy a szerver tűzfala megnyitotta a megfelelő portokat.

## Technikai támogatás

Ha problémába ütközik, ellenőrizze:

- A Docker szolgáltatás megfelelően működik-e
- A hálózati beállítások helyesek-e
- A szerver tűzfal beállításai

## Frissítési módszer

```bash
docker-compose pull
docker-compose up -d
```
