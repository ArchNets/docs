# Szybki przewodnik po wdrożeniu

## 1. Instalacja Dockera

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh ./get-docker.sh
```

## 2. Utworzenie katalogu projektu

```bash
mkdir /opt/archnet && cd /opt/archnet
```

## 3. Utworzenie pliku konfiguracyjnego

### 3.1 Utworzenie `compose.yaml`

```yaml
services:
  archnet-server:
    image: archnet/archnet-server:latest
    container_name: archnet-server
    restart: always
    networks:
      - archnet-network
    ports:
      - 8080:8080
    depends_on:
      archnet-db:
        condition: service_healthy
      archnet-cache:
        condition: service_healthy
    volumes:
      - ./config/archnet.yaml:/app/etc/archnet.yaml

  archnet-db:
    image: mysql:8.4.5
    container_name: archnet-db
    restart: always
    networks:
      - archnet-network
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: archnet
      MYSQL_USER: archnet
      MYSQL_PASSWORD: archnetpassword
    volumes:
      - ./db:/var/lib/mysql
    command:
      - --mysql-native-password=on
      - --bind-address=0.0.0.0
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost', '-uroot', '-prootpassword']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-cache:
    image: redis:7.4.5
    container_name: archnet-cache
    restart: always
    networks:
      - archnet-network
    ports:
      - 6379:6379
    volumes:
      - ./cache:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-admin:
    image: archnet/archnet-admin-web:latest
    container_name: archnet-admin-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3001:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

  archnet-user:
    image: archnet/archnet-user-web:latest
    container_name: archnet-user-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3002:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

networks:
  archnet-network:
    driver: bridge
```

## 4. Uruchomienie usługi

```bash
docker-compose up -d
```

## Informacje o usłudze

### Domyślny adres dostępu

- Panel administratora: `http://adres_IP_serwera:3001`
- Panel użytkownika: `http://adres_IP_serwera:3002`
- API backendu: `http://adres_IP_serwera:8080`

### Domyślne dane logowania

- Email administratora: `admin@archnets.com`
- Hasło początkowe: `password`

### Często używane polecenia zarządzania

```bash
# Sprawdzenie stanu usługi
docker-compose ps

# Sprawdzenie logów usługi
docker-compose logs

# Zatrzymanie usługi
docker-compose down

# Restart usługi
docker-compose up -d
```

## Uwagi

1. Po pierwszym logowaniu natychmiast zmień domyślne hasło.
2. Zaleca się dostosowanie konfiguracji w `archnet.yaml` do rzeczywistych potrzeb.
3. W środowisku produkcyjnym zmień wszystkie domyślne hasła.
4. Upewnij się, że zapora serwera otworzyła odpowiednie porty.

## Wsparcie techniczne

W przypadku problemów sprawdź:

- Czy usługa Docker działa poprawnie
- Czy konfiguracja sieci jest poprawna
- Ustawienia zapory serwera

## Metoda aktualizacji

```bash
docker-compose pull
docker-compose up -d
```

# Wdrożenie jednym kliknięciem

> **Uwagi przed wdrożeniem**
>
> - **Domyślny port panelu administracyjnego to 8080**: Proszę wcześniej skonfigurować rozwiązywanie nazw domen lub upewnić się, że dostępny jest adres IP, aby pomyślnie zakończyć instalację panelu administracyjnego i użytkownika.
> - Serwer: Więcej informacji znajdziesz w [serwerze](/pl-PL/docs/server/config).
> - Panel administracyjny: Więcej informacji znajdziesz w [panelu administracyjnym](/pl-PL/docs/admin/config).
> - Panel użytkownika: Więcej informacji znajdziesz w [panelu użytkownika](/pl-PL/docs/user/config).
