# Rychlý průvodce nasazením

## 1. Instalace Dockeru

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh ./get-docker.sh
```

## 2. Vytvoření adresáře projektu

```bash
mkdir /opt/archnet && cd /opt/archnet
```

## 3. Vytvoření konfiguračního souboru

### 3.1 Vytvoření `compose.yaml`

```yaml
services:
  archnet-server:
    image: archnet/archnet-server:latest
    container_name: archnet-server
    restart: always
    networks:
      - archnet-network
    ports:
      - 8080:8080
    depends_on:
      archnet-db:
        condition: service_healthy
      archnet-cache:
        condition: service_healthy
    volumes:
      - ./config/archnet.yaml:/app/etc/archnet.yaml

  archnet-db:
    image: mysql:8.4.5
    container_name: archnet-db
    restart: always
    networks:
      - archnet-network
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: archnet
      MYSQL_USER: archnet
      MYSQL_PASSWORD: archnetpassword
    volumes:
      - ./db:/var/lib/mysql
    command:
      - --mysql-native-password=on
      - --bind-address=0.0.0.0
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost', '-uroot', '-prootpassword']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-cache:
    image: redis:7.4.5
    container_name: archnet-cache
    restart: always
    networks:
      - archnet-network
    ports:
      - 6379:6379
    volumes:
      - ./cache:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-admin:
    image: archnet/archnet-admin-web:latest
    container_name: archnet-admin-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3001:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

  archnet-user:
    image: archnet/archnet-user-web:latest
    container_name: archnet-user-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3002:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

networks:
  archnet-network:
    driver: bridge
```

## 4. Spuštění služby

```bash
docker-compose up -d
```

## Informace o službě

### Výchozí přístupová adresa

- Administrátorský panel: `http://IP_serveru:3001`
- Uživatelský panel: `http://IP_serveru:3002`
- Backend API: `http://IP_serveru:8080`

### Výchozí přihlašovací údaje

- Email administrátora: `admin@archnets.com`
- Počáteční heslo: `password`

### Časté administrativní příkazy

```bash
# Zobrazit stav služby
docker-compose ps

# Zobrazit logy služby
docker-compose logs

# Zastavit službu
docker-compose down

# Restartovat službu
docker-compose up -d
```

## Poznámky

1. Po prvním přihlášení prosím okamžitě změňte výchozí heslo.
2. Doporučuje se upravit konfiguraci v `archnet.yaml` podle skutečných potřeb.
3. V produkčním prostředí prosím změňte všechna výchozí hesla.
4. Ujistěte se, že je firewall serveru nastaven tak, aby povoloval příslušné porty.

## Technická podpora

Pokud narazíte na problémy, zkontrolujte:

- Zda služba Docker běží správně
- Zda je síťová konfigurace správná
- Nastavení firewallu serveru

## Metoda aktualizace

```bash
docker-compose pull
docker-compose up -d
```

# Jednoduché nasazení

> **Poznámky před nasazením**
>

> - Server: Další informace naleznete v [serverové dokumentaci](/cs-CZ/docs/server/config).
> - Administrátor: Další informace naleznete v [administrátorské dokumentaci](/cs-CZ/docs/admin/config).
> - Uživatel: Další informace naleznete v [uživatelské dokumentaci](/cs-CZ/docs/user/config).
