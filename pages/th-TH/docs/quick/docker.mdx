# คู่มือการติดตั้งอย่างรวดเร็ว

## 1. ติดตั้ง Docker

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh ./get-docker.sh
```

## 2. สร้างไดเรกทอรีโปรเจกต์

```bash
mkdir /opt/archnet && cd /opt/archnet
```

## 3. สร้างไฟล์การตั้งค่า

### 3.1 สร้าง `compose.yaml`

```yaml
services:
  archnet-server:
    image: archnet/archnet-server:latest
    container_name: archnet-server
    restart: always
    networks:
      - archnet-network
    ports:
      - 8080:8080
    depends_on:
      archnet-db:
        condition: service_healthy
      archnet-cache:
        condition: service_healthy
    volumes:
      - ./config/archnet.yaml:/app/etc/archnet.yaml

  archnet-db:
    image: mysql:8.4.5
    container_name: archnet-db
    restart: always
    networks:
      - archnet-network
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: archnet
      MYSQL_USER: archnet
      MYSQL_PASSWORD: archnetpassword
    volumes:
      - ./db:/var/lib/mysql
    command:
      - --mysql-native-password=on
      - --bind-address=0.0.0.0
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost', '-uroot', '-prootpassword']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-cache:
    image: redis:7.4.5
    container_name: archnet-cache
    restart: always
    networks:
      - archnet-network
    ports:
      - 6379:6379
    volumes:
      - ./cache:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-admin:
    image: archnet/archnet-admin-web:latest
    container_name: archnet-admin-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3001:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

  archnet-user:
    image: archnet/archnet-user-web:latest
    container_name: archnet-user-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3002:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

networks:
  archnet-network:
    driver: bridge
```

## 4. เริ่มบริการ

```bash
docker-compose up -d
```

## ข้อมูลบริการ

### ที่อยู่เข้าถึงเริ่มต้น

- แผงผู้ดูแลระบบ: `http://IPของเซิร์ฟเวอร์:3001`
- แผงผู้ใช้: `http://IPของเซิร์ฟเวอร์:3002`
- API ด้านหลัง: `http://IPของเซิร์ฟเวอร์:8080`

### ข้อมูลการเข้าสู่ระบบเริ่มต้น

- อีเมลผู้ดูแลระบบ: `admin@archnets.com`
- รหัสผ่านเริ่มต้น: `password`

### คำสั่งการจัดการที่ใช้บ่อย

```bash
# ดูสถานะบริการ
docker-compose ps

# ดูบันทึกบริการ
docker-compose logs

# หยุดบริการ
docker-compose down

# รีสตาร์ทบริการ
docker-compose up -d
```

## ข้อควรระวัง

1. กรุณาเปลี่ยนรหัสผ่านเริ่มต้นทันทีหลังจากเข้าสู่ระบบครั้งแรก
2. แนะนำให้ปรับแต่งการตั้งค่าใน `archnet.yaml` ตามความต้องการจริง
3. ในสภาพแวดล้อมการผลิตให้เปลี่ยนรหัสผ่านเริ่มต้นทั้งหมด
4. ตรวจสอบให้แน่ใจว่าไฟร์วอลล์ของเซิร์ฟเวอร์เปิดพอร์ตที่เกี่ยวข้อง

## การสนับสนุนทางเทคนิค

หากพบปัญหา กรุณาตรวจสอบ:

- บริการ Docker ทำงานปกติหรือไม่
- การตั้งค่าเครือข่ายถูกต้องหรือไม่
- การตั้งค่าไฟร์วอลล์ของเซิร์ฟเวอร์

## วิธีการอัปเกรด

```bash
docker-compose pull
docker-compose up -d
```

# การติดตั้งแบบ One-Click

> **ข้อควรระวังก่อนการติดตั้ง**
>
> - **พอร์ตเริ่มต้นของแผงผู้ดูแลระบบคือ 8080**: กรุณาตั้งค่าการวิเคราะห์โดเมนล่วงหน้าหรือให้แน่ใจว่ามีที่อยู่ IP ที่ใช้งานได้ เพื่อให้การติดตั้งแผงผู้ดูแลระบบและแผงผู้ใช้เสร็จสมบูรณ์
> - เซิร์ฟเวอร์: ข้อมูลเพิ่มเติมกรุณาอ้างอิงที่ [เซิร์ฟเวอร์](/zh-CN/docs/server/config)
> - แผงผู้ดูแลระบบ: ข้อมูลเพิ่มเติมกรุณาอ้างอิงที่ [แผงผู้ดูแลระบบ](/zh-CN/docs/admin/config)
> - แผงผู้ใช้: ข้อมูลเพิ่มเติมกรุณาอ้างอิงที่ [แผงผู้ใช้](/zh-CN/docs/user/config)
