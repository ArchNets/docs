# Schnellstartanleitung

## 1. Docker installieren

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh ./get-docker.sh
```

## 2. Projektverzeichnis erstellen

```bash
mkdir /opt/archnet && cd /opt/archnet
```

## 3. Konfigurationsdatei erstellen

### 3.1 Erstellen von `compose.yaml`

```yaml
services:
  archnet-server:
    image: archnet/archnet-server:latest
    container_name: archnet-server
    restart: always
    networks:
      - archnet-network
    ports:
      - 8080:8080
    depends_on:
      archnet-db:
        condition: service_healthy
      archnet-cache:
        condition: service_healthy
    volumes:
      - ./config/archnet.yaml:/app/etc/archnet.yaml

  archnet-db:
    image: mysql:8.4.5
    container_name: archnet-db
    restart: always
    networks:
      - archnet-network
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: archnet
      MYSQL_USER: archnet
      MYSQL_PASSWORD: archnetpassword
    volumes:
      - ./db:/var/lib/mysql
    command:
      - --mysql-native-password=on
      - --bind-address=0.0.0.0
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost', '-uroot', '-prootpassword']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-cache:
    image: redis:7.4.5
    container_name: archnet-cache
    restart: always
    networks:
      - archnet-network
    ports:
      - 6379:6379
    volumes:
      - ./cache:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-admin:
    image: archnet/archnet-admin-web:latest
    container_name: archnet-admin-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3001:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

  archnet-user:
    image: archnet/archnet-user-web:latest
    container_name: archnet-user-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3002:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

networks:
  archnet-network:
    driver: bridge
```

### 3.2 Erstellen von `archnet.yaml`

```yaml
Host: 0.0.0.0

  Encoding: plain

  Path: logs
  Level: debug
  MaxContentLength: 0
  Compress: false

  KeepDays: 0
  StackCooldownMillis: 100
  MaxBackups: 0
  MaxSize: 0
  Rotation: daily
  FileTimeFormat: 2025-01-01T00:00:00.000Z00:00
MySQL:
  Addr: archnet-db:3306
  Dbname: archnet
  Username: archnet

  Config: charset=utf8mb4&parseTime=true&loc=Asia%2FShanghai
  MaxIdleConns: 10

  Host: archnet-cache:6379
  Pass:
  DB: 0
Administrator:
  Password: password
  Email: admin@archnet.dev
```

## 4. Dienste starten

```bash
docker-compose up -d
```

## Dienstinformationen

### Standardzugriffsadresse

- Administrationspanel: `http://ServerIP:3001`
- Benutzerpanel: `http://ServerIP:3002`
- Backend-API: `http://ServerIP:8080`

### Standardanmeldedaten

- Administrator-E-Mail: `admin@archnet.dev`
- Anfangspasswort: `password`

### Häufig verwendete Verwaltungsbefehle

```bash
# Dienststatus anzeigen
docker-compose ps

# Dienstprotokolle anzeigen
docker-compose logs

# Dienst stoppen
docker-compose down

# Dienst neu starten
docker-compose up -d
```

## Hinweise

1. Bitte ändern Sie das Standardpasswort sofort nach der ersten Anmeldung.
2. Es wird empfohlen, die Konfiguration in `archnet.yaml` entsprechend den tatsächlichen Anforderungen anzupassen.
3. Ändern Sie alle Standardpasswörter in der Produktionsumgebung.
4. Stellen Sie sicher, dass die Firewall des Servers die entsprechenden Ports geöffnet hat.

## Technischer Support

Bei Problemen überprüfen Sie bitte:

- Ob der Docker-Dienst ordnungsgemäß läuft
- Ob die Netzwerkkonfiguration korrekt ist
- Die Firewall-Einstellungen des Servers

## Upgrade-Methode

```bash
docker-compose pull
docker-compose up -d
```

# Ein-Klick-Bereitstellung

> **Hinweise vor der Bereitstellung**
>
> - **Standardport des Administrationsbereichs ist 8080**: Bitte konfigurieren Sie im Voraus die Domainauflösung oder stellen Sie sicher, dass eine verfügbare IP-Adresse vorhanden ist, um die Installation des Administrations- und Benutzerbereichs erfolgreich abzuschließen.
> - Server: Weitere Informationen finden Sie in der [Serverdokumentation](/de-DE/docs/server/config).
> - Administrationsbereich: Weitere Informationen finden Sie in der [Administrationsdokumentation](/de-DE/docs/admin/config).
> - Benutzerbereich: Weitere Informationen finden Sie in der [Benutzerdokumentation](/de-DE/docs/user/config).
