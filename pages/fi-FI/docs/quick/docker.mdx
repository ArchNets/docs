# Nopea käyttöönotto-opas

## 1. Asenna Docker

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh ./get-docker.sh
```

## 2. Luo projektikansio

```bash
mkdir /opt/archnet && cd /opt/archnet
```

## 3. Luo konfiguraatiotiedosto

### 3.1 Luo `compose.yaml`

```yaml
services:
  archnet-server:
    image: archnet/archnet-server:latest
    container_name: archnet-server
    restart: always
    networks:
      - archnet-network
    ports:
      - 8080:8080
    depends_on:
      archnet-db:
        condition: service_healthy
      archnet-cache:
        condition: service_healthy
    volumes:
      - ./config/archnet.yaml:/app/etc/archnet.yaml

  archnet-db:
    image: mysql:8.4.5
    container_name: archnet-db
    restart: always
    networks:
      - archnet-network
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: archnet
      MYSQL_USER: archnet
      MYSQL_PASSWORD: archnetpassword
    volumes:
      - ./db:/var/lib/mysql
    command:
      - --mysql-native-password=on
      - --bind-address=0.0.0.0
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost', '-uroot', '-prootpassword']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-cache:
    image: redis:7.4.5
    container_name: archnet-cache
    restart: always
    networks:
      - archnet-network
    ports:
      - 6379:6379
    volumes:
      - ./cache:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 3

  archnet-admin:
    image: archnet/archnet-admin-web:latest
    container_name: archnet-admin-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3001:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

  archnet-user:
    image: archnet/archnet-user-web:latest
    container_name: archnet-user-web
    restart: always
    networks:
      - archnet-network
    ports:
      - 3002:3000
    environment:
      NEXT_PUBLIC_API_URL: http://<IP>:8080

networks:
  archnet-network:
    driver: bridge
```

## 4. Käynnistä palvelu

```bash
docker-compose up -d
```

## Palvelutiedot

### Oletusosoite

- Hallintapaneeli: `http://palvelinIP:3001`
- Käyttäjäpaneeli: `http://palvelinIP:3002`
- Taustajärjestelmän API: `http://palvelinIP:8080`

### Oletuskirjautumistiedot

- Hallinnoijan sähköposti: `admin@archnet.dev`
- Alkuperäinen salasana: `password`

### Yleiset hallintakomennot

```bash
# Tarkista palvelun tila
docker-compose ps

# Tarkista palvelun lokit
docker-compose logs

# Pysäytä palvelu
docker-compose down

# Käynnistä palvelu uudelleen
docker-compose up -d
```

## Huomioitavaa

1. Muuta oletussalasana heti ensimmäisen kirjautumisen jälkeen.
2. Suosittelemme säätämään `archnet.yaml` -tiedoston asetuksia tarpeiden mukaan.
3. Tuotantoympäristössä vaihda kaikki oletussalasanat.
4. Varmista, että palvelimen palomuuri on avannut tarvittavat portit.

## Tekninen tuki

Jos kohtaat ongelmia, tarkista:

- Onko Docker-palvelu käynnissä
- Onko verkkokonfiguraatio oikein
- Onko palvelimen palomuurin asetukset kunnossa

## Päivitysmenetelmä

```bash
docker-compose pull
docker-compose up -d
```

# Yhden komennon käyttöönotto

> **Huomioitavaa ennen käyttöönottoa**
>
> - **Hallintapaneelin oletusportti on 8080**: Varmista, että olet määrittänyt verkkotunnuksen tai että sinulla on käytettävissä oleva IP-osoite, jotta hallintapaneelin ja käyttäjäpaneelin asennus onnistuu.
> - Palvelin: Lisätietoja löytyy [palvelimesta](/fi-FI/docs/server/config).
> - Hallintapaneeli: Lisätietoja löytyy [hallintapaneelista](/fi-FI/docs/admin/config).
> - Käyttäjäpaneeli: Lisätietoja löytyy [käyttäjäpaneelista](/fi-FI/docs/user/config).
